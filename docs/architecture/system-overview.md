## Data Flow (Single Turn)

```
User Message
    в”Ӯ
    в–ј
1. CONTEXT MANAGER assembles:
   в”ңв”Җ System Prompt Core (800t)
   в”ңв”Җ Injected Memories (500t)
   в”ңв”Җ Session Summary (400t)
   в”ңв”Җ Turn Buffer (1000t)
   в””в”Җ Current Message
    в”Ӯ
    в–ј
2. PLANNER (if complex request)
   в”ңв”Җ Decomposes goal вҶ’ steps
   в””в”Җ Sends plan to Executor
    в”Ӯ
    в–ј
3. EXECUTOR
   в”ңв”Җ Calls LLM with context
   в”ңв”Җ Handles tool calls via TOOL ROUTER
   в””в”Җ Streams response
    в”Ӯ
    в–ј
4. CRITIC (after response generated)
   в”ңв”Җ Scores response (0.0-1.0)
   в”ңв”Җ If < 0.6 вҶ’ retry with feedback
   в””в”Җ If вүҘ 0.6 вҶ’ deliver
    в”Ӯ
    в–ј
5. REFLECTION ENGINE (async)
   в”ңв”Җ Summarizes exchange
   в”ңв”Җ Extracts facts for memory
   в””в”Җ Writes to long-term memory
    в”Ӯ
    в–ј
6. Response delivered to user
```

## Key Principles

- **Async everywhere**: Memory writes, embedding generation, reflection never block the main response
- **Graceful degradation**: If any subsystem fails, the core chat still works
- **Observability**: Every subsystem emits structured logs
- **Token efficiency**: Context window is assembled with priority ordering
